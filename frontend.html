Definição 2.1

É uma linguagem de estilos (Também não é uma linguagem de programação) usada para descrever como os elementos do documentos serão apresentados (Cor, tamanho, margem, posição, animação etc...). Leia: https://developer.mozilla.org/pt-BR/docs/Web/CSS

O CSS é definido dentro da tag <style>, em um documento .css separado ou como um atributo style="" de um elemento.

Seletores 2.2

Os Seletores definem quais elementos um conjunto de regras CSS se aplica. Nos exemplos anteriores utilizamos um Seletor por tag para definirmos o estilo, mas existem outros tipos de seletores, veja todos:

  • Seletor por tag: Escolhe elementos baseado em sua tag

    Ex: p { ... }

    <h1> Hello World </h1> (Não possui a tag <p>)

    <p> E=mc² </p> (Possui a tag <p>)

  • Seletor por classe: Escolhe elementos baseados no valor de seu atributo class

    Ex: .nome { ... }

    <h1 class="nome"> Hello World </h1> (Possui a class nome)

    <p class="nome"> Lorem Ipsum </p> (Possui a class nome)

    <p> E=mc² </p> (Não possui a class nome)

  • Seletor por ID: Escolhe elementos baseados no valor de seu atributo id

    Ex: #abc { ... }

    <p id="abc"> Hello World </p> (Possui o id abc)

    <p id="xyz"> Hello World </p> (Não possui o id abc)

    Nota

    O atributo id define um identificador exclusivo (ID) que deve ser único por todo o documento.

  • Seletor por atributo: Serve para selecionar todos os elementos com algum outro atributo

    Ex: [type=text] { ... }

    <input type="text"> (Possui type=text)

    <input type="email"> (Não possui type=text)

  • Seletor universal: Serve para selecionar todos os elementos do documento

    Ex: * { ... }

É possível combinar diferentes seletores utilizando os combinadores do CSS:

  • Irmãos Adjacentes - Combinador +: Seleciona o elemento que está imediatamente após o elemento especificado anteriormente
  • Irmãos - Combinador ~: Seleciona o elemento (não necessariamente imediatamente) após o elemento especificado anteriormente
  • Filhos - Combinador >: selecina o elemento que são filhos diretos do elemento especificado anteriormente
  • Descendentes - Combinador (Espaço em branco): selecina o elemento que são filhos do elemento especificado anteriormente (não é necessário que seja um filho direto)

Unidades de medida 2.4

As unidades de medida são meios de mensurar grandezas físicas, tais como metro, litro entre outros. No CSS, mensuramos o milímetro, o pixel, a porcentagem, etc.

No CSS um vasto conjunto de propriedades pode receber valores numéricos relacionados a unidades de medida. São exemplos width, height, margin, padding, border e muitas outras.

Existem dois grupos principais de unidades de medida: Absolutas e Relativas. Veja exemplos:

  • Absolutas:
    • Pixel (px) - Pixel é o menor elemento em um dispositivo de exibição (por exemplo, um monitor), ao qual é possível atribuir-se uma cor. De uma forma mais simples, um pixel é o menor ponto que forma uma imagem digital.
    • Points (pt) - Utilizada por designers, seu uso em geral é para fontes. Exemplo: Arial 12 (O 12 é de 12pt) Frequentemente está relacionada com layouts de impressões e é pouco recomendada para uso em tela.
    • Centímetros (cm) - Unidade de medida que está no nosso dia a dia, porém é pouco usada em telas, uso mais relacionado a layout de impressão também.
    • Milímetros (mm) - Equivalente a décima parte de um centímetro.
    • Inches (in) - Alternativa aos cm e mm é a polegada, mais usada na América do Norte. 1in = 2,54cm
    • Paica (pc) - Unidade herdada da tipografia. De forma geral pouco usada. 1pc = 0,42cm
  • Relativas
    • Ems (em) - Relativo ao tamanho da fonte (font-size) do elemento pai
    • Rems (rem) - Relativo ao tamanho da fonte (font-size) do elemento raiz (no caso de um documento html a tag html)
    • Porcentagem (%) - Relativo ao tamanho do elemento pai
    • Viewport width (vw) - Viewport é a área visível de uma página web para o usuário. 1vw é igual a 1% da largura dessa área vísivel
    • Viewport height (vh) - 1vwh é igual a 1% da altura da área vísivel da página (viewport)

      • Nota

      A diferença entre vw/vh e a % é bem semelhante a diferença entre em e rem, onde a % é relativa ao contexto local do elemento e o vw é relativo ao tamanho total da largura da viewport do usuário.

    • Viewport minimun (vmin) - O vmin é relativo a menor dimensão da viewport, comparando vh e vw. Se 1vh é menor que 1vw o valor de 1vmin será igual a 1vh e vice versa.
    • Viewport maximum (vmax) - O vmin é relativo a maior dimensão da viewport, comparando vh e vw. Se 1vh é maior que 1vw o valor de 1vmax será igual a 1vh e vice versa.
    • x-height (ex) - Relativo a altura do caractere x da fonte utilizada (font-family) (Pouco usada, mais relacionada a alguns ajustes tipográficos mais específicos).
    • Character unit (ch) - Relativo a largura do caractere 0 da font utilizada (font-family) (Pouco usada, mais relacionada a alguns ajustes tipográficos mais específicos).

Ok, mas qual usar? Isso vai depender bastante do contexto, de forma geral opte por Unidades de medida relativas para buscar uma maior responsividade e utilize unidades absolutas para coisas pequenas na tela ou se está criando algo que será impresso. Em telas em e rem são unidades mais recomendadas para questões relacionadas a font-size, margin, padding e media queries, % e vw/vh para width/height e px para coisas pequenas como box-shadows e borders.

Estruturas de dados 3.2

Variável é um dos conceitos mais importantes no estudo de programação, independente da plataforma ou linguagem utilizada. Uma variável referencia a um espaço na memória do computador utilizado para guardar informações que serão usadas em seus programas.

Para elucidar o conceito, imagine que a memória de seu computador é um armário com 100 gavetas e você guarda cada tipo de objeto em uma gaveta diferente. Provavelmente você vai querer criar etiquetas para referenciar o que guarda em cada gaveta.

Mas por que o nome "variável"?

Porque uma variável pode ter o seu valor alterado durante a execução de um programa. Ainda na analogia anterior, suponha que há uma gaveta "Meias" com 2 meias; se necessário, pode-se adicionar mais 5 meias e, caso seja um colecionador, até 200. Assim a gaveta "Meias" muda o seu valor.

Existem casos no programa que determinado dado não deve ser alterado durante a execução, nesse caso se trata de um Constante, outro conceito presente nas linguagens de programação.

Em JS temos uma sintaxe bem simples para declarar variáveis e constantes não sendo necessário referenciar o "tipo" de variável que estamos criando como em outras linguagens de tipagem forte:


let ano = 2022; //Variável

var nome = "Albert"; //Variável Global

const pi = 3.14; //Constante, seu valor não pode ser alterado

let temperatura = -7;

let finalizado = true;

let carro = { marca: "Fiat", cor: "Vermelho", ano: 2011 };

let frutas = ["Maçã", "Uva", "Banana", "Laranja"];

Nota

Valores que são strings devem vir acompanhadas de aspas (").

Números com casas decimais utilize sempre ponto (.) como separador.

Dados booleanos (verdadeiro/falso) você poderá utilizar as palavras reservadas (true) ou (false)
